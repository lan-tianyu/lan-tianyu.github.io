文件属性 及 文件权限

由10个字符组成，第一个字符表明改文件类型，文件类型如下：

值 含义
d 目录
- 普通文件
l 链接文件
b 块文件
c 字符文件
s socket文件
p 管道文件

接下来的属性中，每3个字符为一组，第2-4个字符代表改文件所有者权限，5-7代表文件所有组权限，8-10代表其他用户拥有权限。
每一组都是由rwx组成，r代表只读，w代表可写，x代表可执行，-代表无此类权限

特殊目录 . ..

使用 ls -al 能够看到每个目录下都会存在两个特殊目录(.) (..)，(.)代表当前目录，(..)代表上一层目录。Linux下所有以.开头的文件都是隐藏文件，必须使用 ls -al 才能看到这些隐藏目录。

改变文件权限 chmod

增加权限+，删除权限-，详细权限=
例子：如果是给用户组或者其他用户，则将u改成g或者o
| 作用 | 命令 |
| —— | —— |
| 给某文件添加用户读权限 | chmod u+r somefile |
| 给某文件删除用户读权限 | chmod u-r somefile |
| 给某文件添加用户写权限 | chmod u+w somefile |
| 给某文件删除用户写权限 | chmod u-w somefile |
| 给某文件添加用户执行权限 | chmod u+x somefile |
| 给某文件删除用户执行权限 |chmod u-x somefile端 |
| 添加用户对某文件的读写执行权限 | chmod u+rwx somefile |
| 删除用户对某文件的读写执行权限 | chmod u-rwx somefile |
| 给某文件指定用户拥有读写执行权限 | chmod u=rwx somefile |

递归修改目录及目录下所有子文件及子目录下所有文件的权限，使用 chmod -R xxx somefile

权限掩码：
r-4
w-2
x-1

默认权限umark

Linux系统中的默认权限：

  1. 对于root用户，文件的权限默认为644（rw- -w- -w-），目录的默认权限是755（rwx r-x r-x）
  2. 对于普通用户，文件的权限默认为664（rw- rw- -w-），目录的默认权限是775（rwx rwx r-x）

Linux下，定义目录创建的默认权限值为：umark遮罩777后的权限;定义文件创建的默认权限是：umark遮罩666后的权限。

UID 大于99 设置umark为002。目录权限为777，即 rwx rwx rwx 操作 o-w，最后是rwx rwx r-x，即775；文件权限为666，即 rw- rw- rw- 操作 o-w，最后是rw- rw- r–，即664
UID 小于等于99 设置umark为022，目录权限为777，即 rwx rwx rwx 操作 g-w,o-w，最后是rwx r-x r-x，即755；文件权限为666，即 rw- rw- rw- 操作 g-w,o-w，最后是rw- r– r–，即644

umark 的计算方式不能单纯使用数字同位相减，需要先转化成rwx权限，在进行同为相减，最后转化成数字。比如777-022，得到755，666-022得到644。这是正常的，但是如果是666-011 （rw- rw- rw-）实际计算出来的是（rw- rw- rw-），不是655

查找文件find

find 常见参数

参数 含义
-name filename 查找名为filename的文件
-perm 根据文件权限查找
-user username 根据用户名查找
-mtime -n/+n 查找n天内/n天前更改过的文件
-atime -n/+n 查找n天内/n天前访问过的文件
-ctime -n/+n 查找n天内/n天前创建的文件
-newer filename 查找更改时间比filename新的文件
-type b/d/c/p/l/f/s 查找块/目录/字符/管道/链接/普通/套接字文件
-size 根据文件大小查找
-depth n 最大查找的目录深度

数据库查找 locate

locate依赖Linux数据库文件，Linux系统默认每天会检索系统中所有文件，然后将检索到的文件记录到数据库中，运行locate命令要比find命令反馈更为迅速。执行locate之前，一般要执行updatedb。

sort 排序

举几个例子

sort排序中遇到数字，如果不指定-n，则默认使用第一个数字进行排序

uniq 删除重复内容

uniq在sort后使用，先用sort对文件内容进行排序后，将重复的内容限制在连续的几行中，然后用uniq删除连续重复的内容，仅输出一行。

uniq命令只会对比相邻的行，如果有连续相同的若干行，则删除重复内容留下一行，如果相同的行不连续，就不具备删除效果。

ps 查看进程

ps 命令输出的是当前查询状态下进程瞬间的状态信息

对比 ps -aux 和 ps -ef，如果想要看进程状态，用ps -aux，如果想要获取到父进程id，用ps -ef

top

top 可以查看进程的实时状态信息

top 命令默认输出按照cpu占用率从高大到低进行排序

进程终止kill， killall

kill 后可以使用的信号量一共有64种，最常用的集中有1-重启，9-强行杀掉，15-正常结束

  1. kill -1 pid，为原地重启，重启后进程id不变，其子进程id修改
  2. kill -9 pid，强行杀掉进程，可能导致没清理之前申请的内存，造成内存泄露
  3. kill -15 pid，使进程正常退出

killall [进程名]，eg：killall httpd

killall -h 可以查看参数用法

lsof 查询进程打开文件

编译打包

典型的源码编译安装软件的过程包括以下3步：

  1. configure 结合必要参数生成MakeFile，一般情况下不加也可以生成MakeFile
  2. make 完成自动编译，生成各类模块和主程序
  3. make install 将必要的文件复制到安装目录中
