准备环境

  1. Node.js

上述两个环境直接官网下载对应的安装文件直接安装即可。

Github Page 配置

参考：
超详细Hexo+Github Page搭建技术博客教程【持续更新】

  1. 注册一个Github账号（已经注册过）
  2. 创建一个同名仓库(create a repository)的代码托管一个静态站点：xxx.github.io。一定是与Github账号同名，否则打开是404的。
  3. 配置ssh key（已经配置过）
  4. 部署到Github.

部署到Github

修改部署仓库， 站点根目录下_config.yml，找到 deploy 关键字，修改type关键字value为git，增加github仓库url配置，branch配置：

部署上传，首次上传需要先安装插件

hexo g -d

访问：http://xxx.github.io，首页与http://localhost:4000/一致。

Hexo更改主题

参考：
使用Hexo的NexT主题
hexo的next主题个性化教程:打造炫酷网站
Hexo+NexT搭建博客笔记

简单的配置都是可以在NexT官网中查到对应文档，也可在官网使用search功能查找对应关键字，配置文件_config.yaml 中有提示配置在哪里可以找到doc或者源码或者对应的配置项。

配置关键字在哪里使用？可以使用编辑器打开站点根目录，全局查找某个关键字，一般会在.swig文件中使用，比如footer.swig。

对应关键字数据展示，某些场景写会使用插件函数获取，具体在后面【symbols_count_time 配置】中会提到。

Add copy button on codeblock

copy_button:
enable: false
# Show text copy result.

Follow me on GitHub banner in the top-right corner.

github_banner:
enable: true
permalink: https://github.com/yourname
title: Follow me on GitHub

symbols_count_time setting

Docs: https://theme-next.org/docs/theme-settings/posts.html

symbols_count_time:
symbols: true
time: true
total_symbols: true
total_time: true

Post wordcount display settings

Dependencies: https://github.com/theme-next/hexo-symbols-count-time

symbols_count_time:
separated_meta: true
item_text_post: true
item_text_total: true
awl: 2
wpm: 275

Gitalk

Demo: https://gitalk.github.io

For more information: https://github.com/gitalk/gitalk

gitalk:
enable: true
github_id: yourname # GitHub repo owner
repo: yourname.github.io # Repository name to store issues
client_id: 03526a5c9bcda561ccd6 # GitHub Application Client ID
client_secret: 839dfc9f53d9675bec06d516b4ece751f6aaf117 # GitHub Application Client Secret
admin_user: [] # GitHub repo owner and collaborators, only these guys can initialize gitHub issues
distraction_free_mode: true # Facebook-like distraction free mode
